atulya-zero API Documentation

Modern API System for atulya-zero using FastAPI + Django

Welcome to the atulya-zero API

This API provides access to all atulya-zero functionality through a modern, RESTful interface.
Base URL: http://localhost:8000/api/v1/
Authentication: API Key via X-API-Key header or ?api_key= query param.
Version: 0.1.0

  • All endpoints (except /health and /docs) require authentication.
  • All endpoints return OpenAPI-compliant responses with detailed error codes.
  • See full API documentation for request/response schemas and examples.

Quick Start for Developers

  1. Obtain your API key from the admin or your user dashboard.
  2. Set the X-API-Key header or ?api_key= query parameter in your requests.
  3. Use the endpoints below to interact with Atulya Zero. See request/response examples for each endpoint.
// Example: Send a message
fetch('http://localhost:8000/api/v1/message', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': 'YOUR_API_KEY'
  },
  body: JSON.stringify({ text: 'Hello, Atulya!' })
})
.then(res => res.json())
.then(console.log);

API Endpoints Overview

GET /handlers/

List all available API handlers.
Response: { handlers: string[] }

POST /handlers/{handler_name}

Execute a specific handler with parameters.
Request: { params: object }

POST /message

Send a message to the assistant and receive a response.
Request: { text: string, context?: string }

POST /upload

Upload files to the server.
Request: multipart/form-data

POST /import_knowledge

Import knowledge files and reload memory.
Request: multipart/form-data

POST /transcribe

Transcribe audio using the Whisper model.
Request: { audio: base64, ctxid?: string }

POST /settings_get

Retrieve current system settings.
Response: { settings: object }

POST /settings_set

Update system settings.
Request: { settings: object }

POST /poll

Poll for logs and status updates.
Request: { context?: string, log_from?: number }

GET /health

Health check endpoint to verify API is running.
Response: { status: 'healthy', version: string }

Authentication & Security

  • All endpoints (except /health and /docs) require an API key.
  • API keys can be managed via the Django admin interface.
  • Rate limiting and request logging are enabled for all endpoints.
  • All requests and responses are logged for auditing and debugging.

Error Handling

All errors return a JSON response with success: false, error, and error_code fields. Example:

{
  "success": false,
  "error": "Invalid API key",
  "error_code": "AUTH_ERROR"
}
